# AI-Findir — ИИ-финдиректор для мультиотделового бизнеса

Управленческий аналитический контур: берёт еженедельные Excel-юнит-экономики отделов,
детерминированно считает P&L / калибровку руководителей / обещания, а ИИ-агенты поверх
готовых цифр делают **разбор** и **конкурентный анализ**. На выходе — офлайн-дашборды
для руководителей и собственника.

> Демо-репозиторий: все данные **синтетические** (вымышленная компания «ПримерСерт»).
> Реальные данные в публичную версию не попадают.

## Конвейер узлов

`00 Business Context → 01 Data Extractor → 02 Unit Analyst → 03 Action Analyst →
04 Competitor Analyst → 05 Market/Radar → 06 Action Dashboard`

**Принцип:** цифры считает код (не выдумывает), выводы делает ИИ поверх готовых цифр;
`verify_numbers.py` блокирует любые суммы в тексте агента, которых нет в контракте.

## Быстрый старт

```bash
pip install -r requirements.txt
python make_sample_data.py     # сгенерировать синтетические данные (уже лежат в raw/ и zero_layer.xlsx)
python run_all.py              # собрать все дашборды
```

Открой `out/index.html` — навигация по 4 отделам + свод собственнику (реальный/демо).
Дашборды офлайн-standalone (ECharts вшит), открываются двойным кликом без интернета.

## ИИ-разбор (узел 03) и конкуренты (узел 04)

- **Разбор** пишет LLM-агент через Claude CLI; в `run_all.py` шаг best-effort: нет CLI —
  дашборд всё равно собирается, разбор подставляется при наличии. Офлайн-проверка обвязки: `python run_all.py --stub`.
- **Конкуренты:** `competitor_scan.py` — discovery через Firecrawl (`app.search` ищет активных
  игроков рынка, `app.scrape` собирает их страницы), затем узел 04 фильтрует реальных конкурентов
  и сравнивает. Нужен ключ `FIRECRAWL_API_KEY` в `.env` (в демо не входит).

## Структура

```
run_all.py            единый прогон (одна команда)
depts.py              реестр отделов + авто-поиск свежих xlsx, дата контроля
build_dept.py         extract → контракт analysis.json (цифры)
run_agents.py         LLM-агенты 03/04 поверх контракта (Claude CLI)
analysis_render.py    врезка секций разбора в дашборд
dash.py / owner.py    рендер дашбордов отдела и свода собственнику
competitor_scan.py    discovery + scrape конкурентов (Firecrawl)
make_sample_data.py   генератор синтетических демо-данных
agents_v2/            спецификации ИИ-агентов 03/04/05
skills/findir-dashboard  дизайн-система дашбордов
```
