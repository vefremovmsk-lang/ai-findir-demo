# findir-dashboard — дашборды AI-Findir

Дашборд AI-Findir — это **повестка планёрки**, а не «панель метрик». Читается сверху вниз
за 60 секунд: где горит → почему → кому верить → что оспорить → что делать.
Каждое число обязано существовать в контракте (analysis.json) — рендерер ничего не пересчитывает.

## Жёсткие принципы (нарушение = брак)

1. **Один источник** — рендерер читает только `analysis*.json`. Никаких пересчётов,
   никаких чисел «от себя». Деньги в тексте проверяет `verify_numbers.py`.
2. **Офлайн-standalone** — один HTML-файл, открывается двойным кликом без интернета:
   шрифты только системные, `echarts.min.js` вшивается в файл (vendor/, ~1 МБ).
   Контур закрытый — CDN запрещены.
3. **Герой — тезис, не число.** Не «−2,68 млн», а «−2,68 млн, разложенный по причинам»:
   финрез слева, план-факт мост (или главный график отдела) справа, в одном блоке.
4. **Сигнатура одна.** Смелость — в герое; остальное тихо и дисциплинированно.

## Дизайн-токены

Все токены лежат в `assets/theme_findir.py` — копируй его рядом с генератором и импортируй,
не переопределяй локально. Направление: «печатный финансовый отчёт, оживший на экране».

| Токен | Значение | Роль |
|---|---|---|
| CRIMSON `#d12b3b` | акцент | алерты, провалы, «горит» (фирменный, фирменный красный) |
| INK `#15151a` / PAPER `#faf9f7` | текст/фон | печатная основа |
| INDIGO `#3b4cc0` | данные | нейтральные ряды графиков |
| AMBER / GREEN | `#df9426` / `#2e9e5b` | предупреждение / подтверждено |
| Georgia | заголовки h1/h2, большие числа героя | авторитет годового отчёта |
| Cascadia Mono/Consolas | все числа, eyebrow, таблицы | точность данных |
| Segoe UI | текст | системный, офлайн |

## Анатомия страницы (порядок = повестка планёрки)

```
top (бренд + h1 отдела + мета: дата контроля, источник)
nav  — sticky-якоря по секциям
1. Критичные алерты      (alertband: critical/warn/opportunity)
2. ГЕРОЙ-ТЕЗИС           (финрез + мост/главный график, kpis short-форматом)
3. ⚡ Срочно руководителю (1–3 пункта)
4. ...секции отдела...   (кому верить → воронка/акты → оспорить → SMART → журнал →
                          конкуренты → сезонность → радар → конвейер действий)
последняя секция — ВСЕГДА «Конвейер действий» (действие·основание·ответственный·срок·эффект)
foot — источник + верификация
```

Секции опциональны по данным отдела: нет воронки (СМК) — секция не рендерится, вместо неё
Tier-2 отдела (акты по аудиторам). Заголовки секций — выводы, не ярлыки:
«Кто недодал план», а не «Контрактация».

## Графики — ECharts, паттерны в references/charts.md

Прочитай `references/charts.md` перед созданием любого графика — там проверенные конфиги:
водопад (custom series), калибровка (bar+scatter ромб), воронка (minSize — иначе вырождается),
теплокарта сезонности, «план-призрак» бары. И известные грабли (см. ниже).

## Известные грабли (мы на них уже наступали)

- **Коллизия id**: id графика ≠ id секции (`<div class="sec" id="season">` + график `id="season"`
  → echarts инициализируется на секции, высота 0). Давай графикам суффикс: `seasonhm`, `aud_chart`.
- **JS-функции в option**: `json.dumps` не сериализует функции — вставляй плейсхолдер
  `"formatter": "function"` и заменяй `.replace()` после dumps.
- **₽ в консоли Windows**: print с ₽ падает в cp1251 — в print пиши «руб», ₽ только в файлы.
- **resize**: один обработчик `window.resize` → `echarts.getInstanceByDom(el).resize()` на все .chart.
- **Числа в героe**: только short-формат («2,41 млн»), полные суммы переносятся и ломают сетку.

## Процесс

1. Прочитай контракт отдела (`analysis*.json`) — какие блоки есть, такие секции и рендери.
2. Собери HTML по анатомии выше, импортируя `theme_findir` (CSS, ech_init, money/short/pct).
3. Вшей `vendor/echarts.min.js` одним `<script>`, инициализации графиков — вторым.
4. Самокритика перед сдачей: открой в превью, проверь — все канвасы > 0 высотой,
   KPI не переносятся, тултипы работают, секции в порядке повестки, числа сверены с контрактом.
