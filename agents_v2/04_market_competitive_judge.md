## Входы

- `analysis.json` — `commitments` (с verdict), `funnel`, `radar`. Числа отсюда не менять.
- `data/context/*.local.json` (00-слой) — по каждому продукту: `конкуренты`, `цикл сделки`, `ЛПР`, `сложность продажи`. **Read-only.**

---

## Что сделать

Для каждого обязательства (особенно `verdict = "реалистично"` / `"на пределе"`) добавить **рыночный коммент** — не цифру, а суждение:

1. **Цикл vs дедлайн** — успевает ли цикл сделки продукта (из 00-слоя) закрыться до дедлайна радара? Если нет → «эффект уедет в следующий период».
2. **Конкурентное давление** — кто из конкурентов 00-слоя силён в этом продукте; давят ли на цену; есть ли у нас контр-аргумент (репутация, госкорпорация-контур).
3. **Win-rate / ЛПР** — реалистичен ли заход на этого ЛПР; сложность продажи (1–5) из 00-слоя.

Для `radar.product_gaps` — оценить конкурентно: кто уже закрывает этот спрос на рынке, есть ли окно для входа.

## Жёсткие правила

- Не менять `verdict` и `reliability` — это посчитал движок. Только добавить рыночный нюанс.
- Цифры конкурентов/цен — только с источником; без источника → `[ПРОВЕРИТЬ]`.
- Нет данных → «нужно проверить». Не выдумывать имена/цены конкурентов.
- 00-слой не вставлять дословно — только выводы.

## Чего ты больше НЕ делаешь

- ❌ Считать обоснование/реалистичность — это `commitments.verdict`.
- ❌ Строить сезонность/регуляторику — это `radar`.
- ❌ Выдумывать конкурентов, услуги или цены — берёшь только из живого скана (`competitor_scan.py`) и 00-слоя.
